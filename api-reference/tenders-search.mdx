---
title: "POST /v1/tenders/search"
description: "Search active public tenders by keyword."
---

<ParamField header="X-API-Key" type="string" required>
  Your API key. Requires the `tenders` module.
</ParamField>

## Request body

<ParamField body="keyword" type="string" required>
  Search term to match against tender titles and descriptions (e.g. `"software desarrollo"`).
</ParamField>

<ParamField body="country" type="string" required>
  ISO 3166-1 alpha-2 country code. Supported: `CL`, `BR`, `MX`, `CO`.
</ParamField>

## Response

<ResponseField name="total" type="integer">
  Number of results returned.
</ResponseField>

<ResponseField name="results" type="array">
  List of matching tender objects.

  <Expandable title="Tender object fields">
    <ResponseField name="code" type="string">Tender or process code from the source system.</ResponseField>
    <ResponseField name="name" type="string">Tender title/description.</ResponseField>
    <ResponseField name="status" type="string">Current tender status (varies by country).</ResponseField>
    <ResponseField name="closes_at" type="string">Closing deadline. `null` if unavailable.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="source" type="string">
  Data source: `mercadopublico` (CL), `pncp` (BR), `comprasmx` (MX), `secopii` (CO).
</ResponseField>

<Note>
  Currently returns up to 20 results per page. For Brazil (PNCP) and Mexico (ComprasMX), keyword filtering is applied client-side on the tender description field. For Colombia (SECOP II), keyword filtering is server-side via Socrata SoQL. Pagination is on the roadmap.
</Note>

<RequestExample>

```bash Chile
curl -X POST https://api.enrichlatam.com/v1/tenders/search \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"keyword": "software desarrollo", "country": "CL"}'
```

```python Python
import requests

    "https://api.enrichlatam.com/v1/tenders/search",

)
print(r.json())
```
